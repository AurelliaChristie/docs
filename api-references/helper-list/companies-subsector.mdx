---
title: "Companies by Subsector"
api: "GET /companies/"
description: "Return list of companies from a subsector"
---

### Header

<ParamField header="Authorization" type="string" required>
  Authorization header that should be filled with your Sectors API key.
</ParamField>

### Query Parameters

<ParamField query="sub_sector" type="string" initialValue="insurance" required>
  Subsector from which the list of companies is to be retrieved.
  <AccordionGroup>
    <Accordion title="Get the available sub_sector">
      Get the available `sub_sector` from the [Subsectors](./subsectors)
      endpoint.
    </Accordion>
    <Accordion title="Examples of the sub_sector">
      `bank`, `financing-service`, `insurance`, `retailing`, `tobacco`
    </Accordion>
    <Accordion title="Format">
      The API endpoint only accepts `sub_sector` in the **kebab case** format
      (lowercase and separated by "-").
    </Accordion>
  </AccordionGroup>
</ParamField>

### Response

<ResponseField name="payload" type="string[]">
  Array of ticker of companies in given subsector
</ResponseField>

### Usage

This list can be used for the following endpoint:

- [Company Report](../report/company)

<ResponseExample>
```json 200
[
  "BALI.JK",
  "BTEL.JK",
  "CENT.JK",
]
```

```json 400
[
  "The requested sub_sector does not exist."
]
```

```json 429
{"code": 429, "message": "Rate limit exceeded. Consider upgrading."}
```
</ResponseExample>
